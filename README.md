Ход рыботы
В ходе выполнения лабораторной работы №6 была изучена система контроля версий Git и базовые операции работы с локальным и удалённым репозиторием. Сначала был создан аккаунт на GitHub и выполнен форк репозитория преподавателя. После этого Git был установлен с официального сайта, а затем выполнена настройка имени пользователя и адреса электронной почты. Настройка осуществлялась при помощи команд git config --global user.name и git config --global user.email, результат был проверен через команду git config --list. Скриншоты настройки сохранены в папке img.

После настройки Git локальный репозиторий был создан командой git clone, затем рабочая директория была открыта, и команда git pull подтянула изменения, ранее добавленные через интерфейс GitHub. Скриншоты процесса клонирования и получения изменений также находятся в папке img.

Далее была создана новая ветка при помощи команды git checkout -b feature. В неё были добавлены изменения: в файл README.md была записана строка «Изменение feature», изменения добавлены в индекс командой git add, и выполнена фиксация через git commit -m. Затем работа была продолжена в основной ветке master. В master также был изменён тот же файл README.md, что привело к расхождению версий в двух ветках.

После выполнения изменений в обеих ветках была выполнена команда git merge feature в ветке master. Во время слияния возник конфликт, о чём Git сообщил сообщением «CONFLICT (content): Merge conflict in README.md». Файл README.md был открыт в текстовом редакторе и вручную отредактирован: были удалены служебные строки <<<<<<<, =======, >>>>>>>, а затем оставлены нужные строки. После исправления конфликта файл был добавлен (git add README.md), и выполнена фиксация слияния (git commit -m "Разрешён конфликт при merge feature в master"). Скриншоты возникновения конфликта и его разрешения размещены в папке img.

После успешного слияния ветка feature была удалена командой git branch -d feature. Затем была создана серия небольших изменений в отдельном файле changes.txt. После каждого изменения выполнялись команды git add и git commit -m, что позволило сформировать историю из нескольких последовательных коммитов. После этого был выполнен откат последнего коммита через команду git reset --hard HEAD~1, что вернуло репозиторий к состоянию предыдущего коммита. Скриншоты этих действий также помещены в папку img.

Для оформления отчёта была создана отдельная ветка report командой git checkout -b report. В этой ветке началось формирование итоговой документации, включая логи, команды и скриншоты. История операций была получена в отформатированном виде при помощи команды:
git log --pretty=format:"%h | %ad | %an | %s" --date=short
Лог всех использованных команд (без вывода):

git clone https://github.com/Tramsey00/LR6.git

git add changes.txt
git commit -m "Добавлено второе изменение"

git checkout -b report
git log --pretty=format:"%h | %ad | %an | %s" --date=short
Все скриншоты, иллюстрирующие выполнение шагов лабораторной работы (клонирование, настройка Git, создание веток, конфликт слияния, решение конфликта, удаление ветки, коммиты, откат, создание ветки отчёта), размещены в отдельной папке img в корне репозитория.

Выводы: в результате выполнения лабораторной работы были изучены и применены на практике основные команды Git: работа с удалённым репозиторием, создание и переключение веток, фиксация изменений, просмотр логов, разрешение конфликтов, откат изменений. Также была оформлена документация в формате Markdown, что соответствует требованиям работы.
